What the animation shows

  • Cactus: request enters the primitive → local runs → if confidence < threshold, a single proprietary cloud is called. One path, one cloud, one API key.
  • Ours: request enters a capability router (STT / LLM) → eligibility + score picks top candidate → on low confidence the result is checkpointed and the next candidate is tried → cloud can be the shared OpenRouter backend reused across capabilities.
  • In "Low confidence" mode you can see the cascade step and the checkpoint preservation — a low-conf local answer still returns if the cloud attempt later fails.

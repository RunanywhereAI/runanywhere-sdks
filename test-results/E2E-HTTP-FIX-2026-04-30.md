## Related context from git history (commit `9924540d`)

- Commit `9924540d` ("fix(commons): bundle mbedTLS for Android curl
  HTTPS support") bundles mbedTLS v3.6.2 into `librac_commons.so`,
  giving libcurl a TLS backend on Android (previously the NDK sysroot
  did not ship OpenSSL since r19+, so `find_package(OpenSSL)` returned
  nothing and curl compiled with no TLS backend → `CURLE_UNSUPPORTED_PROTOCOL`).
- This fix predates the H1/H4 work and is orthogonal to it; it is the
  reason libcurl can even attempt TLS on Android at all.
- The verification note in that commit says `rc=77` was the state
  *after* the mbedTLS fix landed, traced to a missing CA bundle — which
  is what H1 intended to address and H4 intended to bypass.

  - H4 (OkHttp transport) is wired correctly in the SDK bridge. If the

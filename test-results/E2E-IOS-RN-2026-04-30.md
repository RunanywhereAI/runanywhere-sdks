## Fix verification

### B05 — `rac_llm_set_stream_proto_callback` buffer ownership (iOS Nitro trampoline)
- **PASS (source-verified)**. `sdk/runanywhere-react-native/packages/core/cpp/HybridLLM.cpp` — the trampoline copies the buffer before handing it to JS:
  ```cpp
  auto buffer = ArrayBuffer::copy(event_bytes, event_size);  // line 38
  try { reg->onBytes(buffer); } catch (...) {}
  ```
  Since the trampoline returns synchronously to the C ABI caller (which then frees its stream buffer after the callback returns), ownership is correctly split: native owns the wire bytes, the Nitro `ArrayBuffer::copy` allocates a JS-owned copy. No `rac_free` is needed in the trampoline because the commons-side caller owns the original `event_bytes`. The registration struct itself is freed in the unsubscribe closure (`delete reg;` at line 83) after `rac_llm_unset_stream_proto_callback`.
- **Not exercisable live** on this device: no LLM model is cached locally and the build has a placeholder Supabase URL, so no download is possible → streaming code path is unreachable without a model. Same blocking situation as Android RN E2E.

### B10 — Embeddings iOS (throws "not implemented" until Nitro regen)

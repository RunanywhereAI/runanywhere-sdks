## Task: repo restructure + de-slop (executed 2026-07-05; all 6 phases done; NOT committed — user reviews, builds, and commits)

### Goal
Clean the repo root: move build systems into their owning folders (like the Kotlin SDK owns its gradle setup), remove AI slop (scratch docs, plan files, banner comments, ascii art, useless md), keep everything working (CI, release, run script, external consumers).

### Root files that CANNOT move (external hard constraints)
- Package.swift + Package.resolved: SwiftPM requires the manifest at the git URL root; all external SPM consumers and release.yml sync-checksums depend on it. Stays.
- jitpack.yml: JitPack only reads repo root. Delete only if JitPack is no longer a Kotlin distribution channel (pending user decision).
- run + run.bat: dev entry point, stays at root; fix its broken commands.
- Dotfile configs (.pre-commit-config.yaml, renovate.json, .gitleaks.toml, .gitignore, .gitattributes, .syncpackrc.json location flexible): tools read them from root.

### Root files that CAN move
- CMake system (CMakeLists.txt, CMakePresets.json, cmake/): move to sdk/runanywhere-commons/ as the new CMake root.
  Option A: commons is CMake root, engines/ and runtimes/ stay siblings, reached via add_subdirectory with out-of-tree paths.
  Option B: also move engines/ and runtimes/ inside sdk/runanywhere-commons/ so commons is fully self-contained (they are commons plugins, nothing else uses them).
  Blast radius: ~11 CI job steps (pr-build native lanes, streaming-perf, release rcli), run script, scripts/build/build-core-android.sh, commons build scripts, wasm build script, rcli CMake root guard. All in-repo, mechanical.
- Root yarn workspace (package.json, yarn.lock, .yarnrc.yml): exists only to link examples/react-native/RunAnywhereAI to the RN SDK packages (RN SDK has its own standalone yarn workspace + lock). Move: make the RN example app its own yarn root with portal: deps to the SDK packages; move the react resolutions pin with it; repoint the rn-typecheck root install step in pr-build.yml.
- dependencies/versions.json + .syncpackrc.json: movable with edits to renovate fileMatch, syncpack config path, and scripts/validation/gates/check_typescript_centralization.sh. Low value, optional.

### Mono scripts/ folder design (proposed)
scripts/
  lib/         shared shell helpers: output formatting, mode detection, version loading
  setup/       environment setup, doctor, toolchain install
  build/       native builds per target: android, ios/xcframework, linux, macos, wasm, windows, rcli
  codegen/     idl proto generation (moved from idl/codegen/)
  release/     version sync, per-sdk packaging, checksums, artifact validation
  validation/  source gates: pii, exports, centralization, deprecated surfaces
run dispatches into these; nothing outside scripts/ except per-component build system files (CMake in commons, gradle in kotlin, package.json per js workspace, pubspec for flutter).

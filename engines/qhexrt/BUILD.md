# QHexRT — Build & Integration Guide

QHexRT is the **private** Qualcomm Hexagon NPU (QNN) backend for the RunAnywhere
SDKs. It runs prebuilt QNN context binaries on Snapdragon Hexagon HTP and is
wired into the Kotlin, Flutter, and React Native SDKs plus a `RunAnywhereAiNPU`
demo app for each.

> **Android arm64 only.** QNN is Snapdragon-only, so there is no iOS / desktop /
> web / armv7 / x86 build. The engine requires a Hexagon **v79 or v81** part
> (Snapdragon 8 Elite / 8 Gen 3 class); older parts fall back / warn.

---

## 1. Repositories required

| Repo | Role | Public? |
|------|------|---------|
| `runanywhere-sdks` (this repo) | Commons, engine glue, SDK packages, demo apps | yes |
| **`QHexRT`** (private, e.g. `~/dev/AI/QHexRT`) | Source of the prebuilt static libs (`libqhexrt_core.a`, `libqhexrt_host.a`) and the C ABI header (`qhexrt_c.h`) | **no — private** |

The QHexRT source never enters `runanywhere-sdks`. Only its compiled `.a`
archives + header are consumed, and even those are **gitignored** (see §3).

## 2. Toolchain prerequisites

- Android SDK + **NDK r27** (`27.0.12077973` / `27.3.x`); export `ANDROID_HOME`
  and `ANDROID_NDK_HOME`.
- CMake ≥ 3.24, Ninja.
- JDK 17, Gradle (wrapper provided).
- Node ≥ 18 + Yarn 3 (Corepack) — React Native only.
- Flutter SDK — Flutter only.
- `protoc` 35.x — only if regenerating protos (see §7).

## 3. Gitignored artifacts (a fresh clone will NOT have these)

Everything below is intentionally excluded from git. After cloning you must
**produce or stage** each item before a build will succeed.

| Artifact | Location | Source / how to regenerate |
|----------|----------|----------------------------|
| `libqhexrt_core.a`, `libqhexrt_host.a` | `engines/qhexrt/prebuilt/lib/arm64-v8a/` | Built by the private **QHexRT** repo (§4.1) |
| `qhexrt_c.h` | `engines/qhexrt/prebuilt/include/qhexrt/` | Copied from the QHexRT repo `include/qhexrt/` |
| `librac_backend_qhexrt.so` (+ `_jni.so` for Kotlin) | each SDK package's `…/jniLibs/arm64-v8a/` | Built from commons + `engines/qhexrt` (§4.2) and staged (§4.3) |
| `librac_commons.so` | each core package's `…/jniLibs/arm64-v8a/` | Built from commons (§4.2); must contain the new `rac_npu_probe` symbol (§5) |
| `build/`, `node_modules/`, `.cxx/`, `.gradle/`, app `*/build/` | everywhere | regenerated by the build |

The `.so` files are bundled (not downloaded from a GitHub release like the
public llamacpp/onnx backends) because QHexRT is private — but they stay out of
git, so they are staged locally per the steps below.

## 4. Build pipeline

### 4.1 Build the QHexRT static libs (private repo)

In the private QHexRT checkout, build the Android arm64 static libraries with
its own CMake (Android toolchain, NDK r27). This produces:

```
libqhexrt_core.a
libqhexrt_host.a
include/qhexrt/qhexrt_c.h
```

Then stage them into this repo (the path `engines/qhexrt` CMake defaults to via
`QHEXRT_ROOT`, or pass `-DQHEXRT_ROOT=<path>`):

```bash
# from runanywhere-sdks root; adjust QHEXRT to your private checkout
QHEXRT=~/dev/AI/QHexRT
mkdir -p engines/qhexrt/prebuilt/include/qhexrt engines/qhexrt/prebuilt/lib/arm64-v8a
cp "$QHEXRT"/include/qhexrt/qhexrt_c.h        engines/qhexrt/prebuilt/include/qhexrt/
cp "$QHEXRT"/<build>/libqhexrt_core.a "$QHEXRT"/<build>/libqhexrt_host.a \
     engines/qhexrt/prebuilt/lib/arm64-v8a/
```

If the prebuilt tree is absent the `engines/qhexrt` CMake degrades gracefully
(the backend is skipped), so the rest of commons still builds.

### 4.2 Build commons + the QHexRT engine for Android arm64

```bash
export ANDROID_NDK_HOME=$ANDROID_HOME/ndk/27.0.12077973
./run build-android            # wraps `cmake --preset android-arm64` (+ build)
# or directly:
cmake --preset android-arm64 && cmake --build --preset android-arm64
```

Outputs land in `build/android-arm64/`:

```
build/android-arm64/sdk/runanywhere-commons/librac_commons.so   # includes rac_npu_probe
build/android-arm64/engines/qhexrt/librac_backend_qhexrt.so
build/android-arm64/engines/qhexrt/librac_backend_qhexrt_jni.so  # Kotlin JNI bridge
```

### 4.3 Stage the natives into each SDK package

Copy the freshly built `.so` into the package `jniLibs` (arm64-v8a). The
**core** `librac_commons.so` must be re-staged too — the one published with the
older SDKs lacks `rac_npu_probe`, which silently breaks NPU detection.

```bash
A=build/android-arm64
# --- engine backend .so into each qhexrt package ---
cp $A/engines/qhexrt/librac_backend_qhexrt.so \
   sdk/runanywhere-kotlin/modules/runanywhere-core-qhexrt/src/main/jniLibs/arm64-v8a/
cp $A/engines/qhexrt/librac_backend_qhexrt_jni.so \
   sdk/runanywhere-kotlin/modules/runanywhere-core-qhexrt/src/main/jniLibs/arm64-v8a/
cp $A/engines/qhexrt/librac_backend_qhexrt.so \
   sdk/runanywhere-flutter/packages/runanywhere_qhexrt/android/src/main/jniLibs/arm64-v8a/
cp $A/engines/qhexrt/librac_backend_qhexrt.so \
   sdk/runanywhere-react-native/packages/qhexrt/android/src/main/jniLibs/arm64-v8a/

# --- fresh commons (with rac_npu_probe) into EACH core package ---
cp $A/sdk/runanywhere-commons/librac_commons.so \
   sdk/runanywhere-flutter/packages/runanywhere/android/src/main/jniLibs/arm64-v8a/
cp $A/sdk/runanywhere-commons/librac_commons.so \
   sdk/runanywhere-react-native/packages/core/android/src/main/jniLibs/arm64-v8a/
# (Kotlin core commons is refreshed by its own publish/build.)
```

Verify the symbol made it in:

```bash
$ANDROID_NDK_HOME/toolchains/llvm/prebuilt/*/bin/llvm-nm -D \
  sdk/runanywhere-react-native/packages/core/android/src/main/jniLibs/arm64-v8a/librac_commons.so \
  | grep rac_npu_probe        # expect one hit
```

> **Why per-target:** each SDK target bundles its *own* copy of
> `librac_commons.so`. Adding a commons symbol means re-staging the fresh `.so`
> into every target, not just one.

### 4.4 Build the existing example apps (they consume QHexRT)

QHexRT is consumed by the **existing** `RunAnywhereAI` example apps — there is no
separate NPU app. Each registers the QHexRT backend (Android-only, guarded
alongside llamacpp/onnx), surfaces the NPU capability probe, and exposes a
dedicated NPU section.

**Kotlin** (`examples/android/RunAnywhereAI`):

```bash
./run example android build      # stage SDK AARs, then assembleDebug
./run example android install    # + installDebug + launch
```

**Flutter** (`examples/flutter/RunAnywhereAI`):

```bash
cd examples/flutter/RunAnywhereAI
flutter pub get
flutter build apk --debug
flutter install
```

**React Native** (`examples/react-native/RunAnywhereAI`):

```bash
# from repo root — links the workspace (includes @runanywhere/qhexrt)
corepack yarn install
# regenerate the Nitro bridge if the .nitro.ts spec changed:
(cd sdk/runanywhere-react-native/packages/qhexrt && \
   ../../node_modules/.bin/nitrogen)

cd examples/react-native/RunAnywhereAI/android
./gradlew :app:assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk

# dev (Metro): from the app dir
adb reverse tcp:8081 tcp:8081
corepack yarn start
```

## 5. NPU capability detection

`rac_npu_probe` (commons, `rac_npu_capability.{h,cpp}`) reads the SoC identity
(`ro.soc.model`, `/sys/devices/soc0/soc_id`) **without loading QNN** and reports
the Hexagon arch + whether QHexRT supports it (v79/v81). Each demo app calls it
on launch (`QHexRT.probeNpu()`) and warns on unsupported devices. Verified
on-device result on an `SM7635`: `Hexagon v73`, unsupported → warning shown,
backend still registers.

## 6. Build flow at a glance

```
QHexRT (private)  ──build──▶  libqhexrt_*.a + qhexrt_c.h
        │ stage
        ▼
engines/qhexrt/prebuilt/  ──┐
                            ├─ cmake --preset android-arm64 ─▶ librac_commons.so
sdk/runanywhere-commons/  ──┘                                  librac_backend_qhexrt(.so/_jni.so)
        │ stage .so into each package jniLibs
        ▼
kt / flutter / rn  qhexrt packages + core packages
        │ consumed by
        ▼
existing RunAnywhereAI example apps  (kt / flutter / rn)
```

## 7. Known gaps

- **No model bundles published yet.** The Models screen catalog is empty until
  QHexRT/NPU bundles land in the npu-forge HuggingFace repo. Download/load flows
  are wired and ready.
- **`@runanywhere/proto-ts` not regenerated.** The IDL/commons/Kotlin carry
  `INFERENCE_FRAMEWORK_QHEXRT = 24`, but the React Native TS wire types do not
  yet. Detection and builds are unaffected; QHEXRT model registration over the
  RN bridge needs proto-ts regenerated (`protoc` 35.x) once model links exist.
